---
title: Import Mode
description: Convert a stack to a .xstk package.
---

Import mode is the default mode:

```sh
stackimport import [--nodumprawblocks] [--dumprawblocks] [--nostatus] [--noprogress] [--rawgraphics] [--output <packagePath>] [--media-root <dir>] <originalStackPath>
```

For compatibility, the input path can still be supplied without the `import`
subcommand.

If `--output` is omitted, StackImport writes a sibling `.xstk` directory next to
the input stack.

| Option                   | Behavior                                                      |
| ------------------------ | ------------------------------------------------------------- |
| `--output <packagePath>` | Write the generated package to an explicit directory.         |
| `--dumprawblocks`        | Write raw `.data` files for stack blocks. Enabled by default. |
| `--nodumprawblocks`      | Skip raw block exports for smaller packages.                  |
| `--nostatus`             | Suppress status messages.                                     |
| `--noprogress`           | Suppress progress messages.                                   |
| `--rawgraphics`          | Export raw bitmap data instead of decoded graphics.           |
| `--media-root <dir>`     | Package loose external media from a nearby directory.         |

The package contains JSON metadata, decoded media where supported, raw resource
data where preservation is safer than conversion, and raw block evidence by
default.

Every import writes `source-manifest.json`. For classic Mac files, the manifest
records Finder type/creator, data fork SHA-256, resource fork byte count,
resource fork SHA-256 when available, and a coarse source classification such as
`hypercard_stack` or `hypercard_standalone_application`.

Imports also write `script-index.json`, a heuristic HyperTalk index for
reverse-engineering. It lists script owners, handlers, string literals,
external-looking calls such as XCMD/XFCN or HyperTint calls, and likely
card/stack/movie/sound/picture references inferred from script context.

When `--media-root` is provided, StackImport copies regular files under that
directory into `external-assets/files/` in the `.xstk` package and writes
`external-assets/manifest.json`. The manifest records source path, package path,
byte size, SHA-256, Finder type/creator when available, extension, coarse asset
kind, and media type. The source stack itself, `.xstk` directories, and
`.DS_Store` files are skipped.

QuickTime movie assets also receive `external-assets/quicktime-analysis/*.json`
reports from the vendored `mov2qt` parser. These reports record data-fork atoms,
classic resource-fork `moov` atoms, movie timing, track timing, sample counts,
chunk counts, and sample description formats such as `cvid`, `rpza`, `rle `,
`raw `, and `twos`. Each sample description includes codec mapping fields for
the codec name, family, decoder status, permissive-decoder status, and preferred
modern output.
