---
title: CLI Overview
description: Major stackimport command line modes.
---

The `stackimport` executable has three direct modes and one repository-level
batch workflow. The preferred spelling uses subcommands; legacy `--scan` and
`--rom` flags remain supported.

| Mode   | Command shape                                                      | Purpose                                                    |
| ------ | ------------------------------------------------------------------ | ---------------------------------------------------------- |
| Import | `stackimport import [options] <stack>` or `stackimport <stack>`    | Convert a HyperCard stack to a `.xstk` package.            |
| Scan   | `stackimport scan <stack>`                                         | Inspect the block stream without importing.                |
| ROM    | `stackimport rom [--rom-base <address>] [--emit-atlas] <rom>`      | Analyze classic Mac ROMs and optionally emit atlas TSVs.   |
| Corpus | `scripts/import_all_stacks.py --stackimport-bin build/stackimport` | Batch extract, classify, import, and index stack archives. |

Build the CLI with:

```sh
cmake -S . -B build
cmake --build build --target stackimport
```

Use `stackimport --help` or `stackimport <mode> --help` for generated option
help.
